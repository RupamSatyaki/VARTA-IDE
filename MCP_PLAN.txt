  WHAT IS MCP IN VARTA?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  MCP (Model Context Protocol) is a tool execution layer that allows the AI
  assistant to directly interact with the IDE — creating files, running terminal
  commands, making git commits, searching code, and much more.

  Think of it as giving the AI "hands" inside the editor. Instead of just
  suggesting code, the AI can actually write it, run it, and commit it.

  Key Principles:
  - Every tool is sandboxed and permission-gated
  - Destructive operations require user confirmation
  - All tool calls are logged and undoable
  - External MCP servers (Claude Desktop, etc.) can connect
  - Tools stream results in real-time

  TOOL DEFINITION INTERFACE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Every tool follows this standard contract:

  interface ToolDef<TInput, TOutput> {
    name:        string           // e.g. "filesystem.readFile"
    description: string           // Human-readable, used by AI
    category:    ToolCategory     // filesystem | editor | terminal | git | ...
    permission:  Permission       // read | write | execute | admin
    inputSchema: ZodSchema        // Validated before execution
    streaming?:  boolean          // Supports AsyncGenerator output
    dangerous?:  boolean          // Requires user confirmation
    execute(input: TInput, ctx: MCPContext): Promise<TOutput>
              | AsyncGenerator<TOutput>
  }

  Permission Levels:
  ┌─────────────┬──────────────────────────────────────────────────────────┐
  │ read        │ filesystem read, search, git status, workspace info      │
  │ write       │ filesystem write, git stage/commit, editor changes       │
  │ execute     │ terminal commands, git push/pull, format document        │
  │ admin       │ settings change, folder open, install tools              │
  └─────────────┴──────────────────────────────────────────────────────────┘

  KEY IMPROVEMENTS OVER BASIC MCP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. STREAMING RESULTS
     Long-running tools (terminal, search) stream output in real-time
     using AsyncGenerator. User sees progress, not just final result.

  2. TOOL CHAINING / PIPELINES
     Chain tools: searchInFiles → readFile → applyDiff → commit
     Pre-built pipelines for common workflows.
     Visual pipeline builder in UI.

  3. TOOL CALL HISTORY + UNDO  ← Unique to Varta
     Every destructive tool call takes a snapshot before executing.
     User can undo any AI action from the MCP log viewer.
     This is NOT available in VS Code Copilot or Cursor.

  4. SANDBOXING
     PathGuard: AI cannot access files outside workspace root.
     CommandFilter: Blocks rm -rf, format C:, etc.
     ConfirmationGate: User must approve dangerous operations.

  5. EXTERNAL MCP SERVER SUPPORT
     .varta/mcp.json config — connect to any MCP-compatible server.
     Compatible with Claude Desktop, Cursor, and community servers.
     Database tools, browser tools, API tools — all pluggable.

  6. RESULT CACHING
     Read-only tools cache results with TTL.
     Write operations automatically invalidate relevant cache entries.
     Reduces redundant AI tool calls, improves response speed.

  7. TOOL MARKETPLACE
     Community tools installable via npm.
     Enable/disable tools per workspace.
     Settings panel "MCP Tools" tab.
